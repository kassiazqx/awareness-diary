# Lessons Learned

从旧项目迁移来的精简教训：

1. 状态会因组件卸载丢失；跨页面状态必须提前决定 owner。
2. 副作用不能散落在页面，AI 调用、保存、同步都要有统一入口。
3. DB 更新不会自动更新内存状态；保存后谁读新数据要提前设计。
4. 多入口会导致顺序不一致；新增入口前必须对照旧入口。
5. 多对多关系越多，删除、同步、导出越容易漏。
6. localStorage / DB / 内存不能同时像真源。
7. 未来能力不能只进 schema 不接运行时；停放能力必须登记。
8. 默认数据初始化只能有一个 owner。
9. AI 输出不可完全信任，解析和清理必须容错。
10. 本地优先项目从第一天就要为同步留下 ID、时间戳、软删除、冲突接口。
